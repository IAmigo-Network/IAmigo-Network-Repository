1. Requisitos Específicos
1.1 Prioridades dos Requisitos
Para estabelecer a prioridade dos requisitos, foram adotadas as denominações: essencial, importante e desejado. Abaixo temos a descrição de significado de cada uma dessas denominações:
Essencial: É o requisito sem o qual o sistema não entra em funcionamento. Requisitos essenciais são requisitos imprescindíveis, que têm que ser implementados impreterivelmente.
Importante: É o requisito no qual o sistema entra em funcionamento, mas de forma não satisfatória. Requisitos importantes devem ser implementados, mas se não forem, o sistema poderá ser implementado e usado mesmo assim.
Desejável: É o requisito que não compromete as funcionalidades básicas do sistema, isto é, o sistema pode funcionar de forma satisfatória sem ele. Requisitos desejáveis são requisitos que podem ser deixados para versões posteriores do sistema, caso não haja tempo hábil para implementá-los na versão que está sendo especificada.
1.2 Requisitos Funcionais (RF) e Regras de Negócio (RN)
RF01: Cadastro e Perfil do Usuário
Prioridade: Essencial | Autor: Allan Puga
Descrição: O sistema deve possuir um módulo central de cadastro de usuário, permitindo o armazenamento de dados pessoais e de personalização.
Regras de negócio:
RN01 - Nível de Assistência: No cadastro, o sistema deve permitir selecionar o "Perfil de Necessidade" (Ex: TDAH, Melhor Idade, Rotina Intensa), o que mudará automaticamente a complexidade visual do sistema.
RN02 - Vínculo de Perfis e Aceite: O sistema deve gerar um código único (Token) no perfil do assistido. Um perfil de "Cuidador" só poderá visualizar os dados e receber alertas de emergência após o aceite de um convite formal via e-mail ou link de segurança.
RN03 - Preferência de Notificação: O usuário deve escolher o canal principal de alerta (WhatsApp, E-mail ou Notificação no App).
RF02: Gerenciador de Tarefas e Compras
Prioridade: Essencial | Autor: Allan Puga
Descrição: O sistema deve fornecer um gerenciador de tarefas que permita ao usuário criar novas atividades, editar as existentes, gerenciar listas de compras e marcá-las como concluídas.
Regras de negócio:
RN04 - Quebra de Tarefas (Micro-steps): Tarefas longas devem permitir a criação de "subtarefas" para evitar sobrecarga cognitiva.
RN05 - Sistema de Recompensas (Gamificação): Ao concluir uma tarefa dentro do prazo, o usuário ganha "pontos de foco" ou uma animação comemorativa na tela.
RN06 - Recorrência Inteligente: Tarefas devem ter opções de repetição (diária, dias úteis, semanal), sem que o usuário precise recriá-las manualmente.
RN07 - Auto-limpeza da Lista de Compras: Itens marcados como "comprados" devem ser movidos para o histórico após 24 horas, para manter a lista limpa e organizada.
RF03: Sistema de Lembretes, Agenda e Financeiro
Prioridade: Essencial | Autor: Allan Puga
Descrição: O sistema deve contar com um módulo de lembretes voltado para a estruturação da rotina diária, controle de medicação, alertas de contas e visualização de agenda.
Regras de negócio:
RN08 - Confirmação de Dose: O sistema só deve considerar uma medicação como "concluída" após o clique no botão "Já Tomei". Se o usuário apenas fechar a notificação, o status permanece como "Pendente".
RN09 - Escalonamento de Alerta de Medicação: Caso a medicação não seja confirmada em até 15 minutos, o sistema dispara um segundo alerta (reforço). Se após 30 minutos o usuário ainda não tiver confirmado (ou ignorar 3 vezes seguidas no botão "Soneca"), um alerta automático deve ser enviado ao WhatsApp do Cuidador vinculado.
RN10 - Registro de Atraso: O sistema deve registrar o horário real da confirmação da medicação para gerar gráficos de adesão ao tratamento.
RN11 - Validação via WhatsApp: O sistema deve ser capaz de ler respostas simples no WhatsApp (Ex: "Digite 1 para confirmar que tomou o remédio ou 2 para lembrar em 10 min").
RN12 - Bloqueio de Conflitos: O sistema deve avisar se o usuário tentar marcar um compromisso no mesmo horário de outro já existente.
RN13 - Antecedência de Avisos: Lembretes de consultas médicas e contas a vencer devem ser enviados com 24 horas de antecedência e novamente 2 horas antes do evento.
RN14 - Baixa em Contas: Uma conta cadastrada só sai da lista de "Pendentes" após o usuário anexar o comprovante (opcional) ou marcar manualmente como "Pago".
RF04: Dashboard de Acompanhamento
Prioridade: Importante | Autor: Allan Puga
Descrição: O sistema deve apresentar um painel (dashboard) que consolide as informações, alertas urgentes e métricas de uso do assistente.
Regras de negócio:
RN15 - Priorização Visual: O Dashboard deve exibir no topo, de forma destacada, as tarefas, contas e remédios que estão atrasados ou que vencem na próxima 1 hora.
RN16 - Modo Foco Restritivo: Ao ativar o Modo Foco, a interface deve ocultar indicadores financeiros e listas, mantendo visível apenas a tarefa atual e o cronômetro (Pomodoro).
RN17 - Relatório de Humor/Energia: Ao iniciar o dia, o dashboard pode perguntar "Como você está se sentindo hoje?", registrando essa métrica para relatórios de saúde.
RN18 - Código de Cores e Acessibilidade: O sistema deve manter o padrão visual: Verde (Saúde/Remédios), Azul (Compromissos), Laranja (Tarefas) e Vermelho (Financeiro). O dashboard também deve ter opções de Modo Noturno e Alto Contraste para daltônicos.
RN19 - Visualização de Relatórios do Cuidador: O cuidador terá acesso à visualização de tarefas e remédios do assistido, mas não poderá excluir registros históricos para garantir a integridade dos dados de saúde.
RF05: Assistente Inteligente (Chatbot)
Prioridade: Desejável | Autor: Allan Puga
Descrição: O sistema deve incluir uma interface de assistente inteligente operando no formato de chatbot.
Regras de negócio:
RN20 - Criação de Tarefa por Áudio: O chatbot no WhatsApp deve aceitar mensagens de áudio, transcrevê-las e convertê-las em uma tarefa estruturada no sistema.
RN21 - Briefing Matinal: Todo dia em um horário pré-definido, o chatbot envia um resumo do dia informando tarefas e medicações agendadas.
RN22 - Humanização por IA e Tom de Voz: As mensagens devem passar por um filtro de IA para que o tom seja empático, não mecanizado e não punitivo (Ex: ao invés de "Você atrasou", usar "Notei que essa tarefa ficou para trás, quer reagendar?").
RN23 - Aprendizado de Rotina: A IA deve analisar os horários reais de conclusão de tarefas por 7 dias antes de começar a sugerir ajustes automáticos inteligentes na agenda do usuário.
1.3 Requisitos Não Funcionais (RNF)
RNF01: Integração de Comunicação
Prioridade: Essencial | Autor: Allan Puga
Descrição: O envio de mensagens deve ser obrigatoriamente integrado utilizando uma API oficial ou suportada do WhatsApp.
RNF02: Armazenamento de Dados
Prioridade: Essencial | Autor: Allan Puga
Descrição: Os dados da aplicação, rotinas e usuários devem ser persistidos no banco de dados Firebase Firestore (conforme tecnologia obrigatória do projeto).
RNF03: Disponibilidade e Desempenho
Prioridade: Importante | Autor: Allan Puga
Descrição: O agendamento e disparo das mensagens deve ter alta precisão, garantindo que os lembretes não sofram atrasos superiores a 1 minuto do horário programado.
RNF04: Segurança e Privacidade (LGPD)
Prioridade: Essencial | Autor: Allan Puga
Descrição: O sistema deve garantir a criptografia de dados sensíveis e estar em conformidade com a LGPD para o tratamento e compartilhamento de informações de saúde com cuidadores.
